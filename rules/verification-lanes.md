# Lane-таблица: поверхность фичи → обязательное доказательство (v6.2 F5)

> Урок П2 аудита: «зелёные юнит-тесты лгут» — менеджер не мог выдать промокод при passing-статусе.
> Доказательство обязано соответствовать ПОВЕРХНОСТИ фичи — тому месту, где её видит пользователь.
> Hook state-transition проверяет наличие evidence по surface; эта таблица — что именно класть.

| surface | Что запустить | Какое evidence обязано лечь в фичу | НЕ считается |
|---|---|---|---|
| **ui** | браузер (headless e2e или живой прогон) на целевых устройствах | `layer_4_user_at`/`layer_5_user_at` + путь к скриншоту/прогону: элемент виден, нажимается, результат наступил | юнит-тесты, «компилируется», curl на страницу |
| **api** | реальный вызов точки: curl/HTTP-клиент | команда + статус-код + ключевое поле ответа (вставить в evidence) | мок-тест обработчика без сетевого вызова |
| **cli** | точная команда, как её запустит пользователь | команда + exit code + первые строки вывода | прямой вызов внутренней функции |
| **job** | реальный (или вручную триггернутый) прогон фоновой задачи | лог прогона: запустился → обработал → результат лёг куда надо | «функция написана», расписание создано |
| **service** | behavior-probe: послать запрос/сообщение — получить ответ | вывод probe (запрос → ответ за N сек) | `pgrep`/PID-проверка («процесс жив» ≠ «отвечает» — бот молчал 5 дней при живом PID) |
| **lib / logic** | прогнать логику (юнит/интеграция/e2e) | `layer_2_runtime_at` и выше — след РЕАЛЬНОГО прогона (не только `layer_1_syntax`); typecheck+lint не ловят разъехавшееся поведение (v8 L5-F2) | «компилируется», только layer_1_syntax |
| **content** | просмотр результата | layer_1..2 timestamps достаточно | — |

**Negative-gate обязателен на M/L (v8 L5-F2, c9):** любая M/L-фича в passing обязана нести `verification_self_check.negative_test_at` (mutation — искусственный баг роняет тест) ИЛИ `leak_check_at` (expected не в одном файле с input). Без него «зелёные тесты лгут». Hook state-transition блокирует M/L passing без negative-gate. S — light path, освобождена.

## Live-Target Probe — что делать, если «не могу прогнать живьём» (паттерн pilot-shell)

Перед тем как честно признать «e2e невозможен», пройди 4 яруса (каждый — команда + её итог):

1. **Найти уже работающую цель**: `curl -s localhost:3000/health` / `lsof -i :3000` —
   dev-сервер уже может быть поднят.
2. **Поднять самому**: запусти dev-сервер (`npm run dev` / `python -m ...`) в фоне,
   подожди health до 60 сек.
3. **Preview-деплой**: в проекте есть vercel.json / fly.toml / Dockerfile → подними preview
   и прогони на нём.
4. **Только после задокументированного провала 1-3** — фиксируй честный статус
   `UNIT_VERIFIED` (НЕ `passing`): в evidence напиши, какие ярусы пробовал и что упало.
   Skip молча — не вариант: это ровно тот путь, которым «готово» доезжало до пользователя сломанным.
