# Роутинг модели по тиру + защитная безопасность (v8 L1-F2 / L1-F4)

Контракт «роль → модель → усилие» — единый источник `docs/agent-registry.md` (сверяется
`scripts/check-plugin-self.sh` раздел 4: модель фронтматтера обязана совпадать с реестром).

## Тиры (L1-F2)

- **Opus** — планирование / детализация / архитектура / критика / проверка / оценка. Дорогой
  reasoning там, где цена ошибки высока.
- **Sonnet** — написание кода / исполнение / чтение-сырья / механические merge и сортировки.
  Дешёвый исполнитель по детальному плану: главную ошибку ловят механизмы harness
  (4-слойная проверка, границы правок, evidence-гейт, circuit breaker), а не ум модели.
- **effort = max** во всех ролях (решение владельца: качество приоритетно, экономия — через
  выбор модели, не через понижение усилия).

Смена привязки роль→модель — правкой реестра + фронтматтера (одним движением). Это **смена
контракта** (влияет на 100% выводов роли) — перед выкаткой прогон на 1–2 реальных сценариях
(правило dependency-swap; носитель — `hooks/checks/model-swap-guard.sh`).

## Защитная безопасность → Opus, НЕ свежайшая frontier (L1-F4)

Защитную / security-работу — написание security-хуков и защитных гейтов, разбор уязвимостей,
харденинг, анализ вредоносного ввода — роутить на **Opus**, а не на свежайшую frontier-модель.

**Почему:** safety-классификаторы самых свежих frontier-моделей склонны отказывать в benign
defensive-security работе mid-task (ложно принимают её за атакующую) — задача обрывается на
середине. Opus стабильно выполняет защитную работу. Урок pilotfish.

**Честная граница (discipline):** надёжного хука на «смысл задачи = security» нет — нельзя
машинно отличить защитную правку от обычной. Носители правила: этот файл + предупреждение
`model-swap-guard` при смене модели + строка в `docs/traceability.md`. Не enforcement-механизм,
а дисциплина; дополняет dependency-swap-контракт.

**Пример из самого плагина:** must-fix L5-F1 (защита границ правок) делался на Opus в main-session,
а не делегировался дешёвому тиру — ровно по этому правилу.
