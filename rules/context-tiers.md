# Три уровня контекста (v8 L4-F1)

> Закрывает c3 (рост файлов) и c8 (не полагаться на авто-сжатие). Разведка: Anthropic
> («Effective context engineering») и Manus трактуют авто-сжатие как **крайнюю меру у самого
> лимита** (raw → compaction → summarization), а не рабочий инструмент. Правильная цель —
> **малый горячий контекст + состояние в файлах**, тогда порог ~95% почти не достигается и
> «рулетка сжатия» не запускается. Управляем сами (`/checkpoint`), а не отдаём движку.

## Три уровня

| Уровень | Что там | Где | В контексте? |
|---|---|---|---|
| **Горячий** | Активные + up_next фичи (6-строчная провенанс-голова) + **индекс архива** (одна строка на завершённую) | `CLAUDE.md` ≤200 строк, `SESSION.md` → Current State | **всегда** (грузится автоматически) |
| **По требованию** | Детали фич, backlog, журнал, `docs/decisions/`, `docs/research/`, provenance-голова истории | `feature_list.json` тело, `docs/`, `error-journal.md` | по `grep`/`offset`, точечно |
| **Холод** | Завершённое (тело+доказательство), старый журнал, полная история требований, архив | `feature_list.archive.json`, `.harness/provenance-log.jsonl` | **никогда** в CLAUDE.md-контекст; только по явной ссылке |

## Правило горячего контекста

- `CLAUDE.md` — роутинг ≤200 строк. Не тело фич, а карта: где что лежит.
- Завершённая (архивная) фича в горячем = **одна строка-индекс** (`feat-012 → archive#feat-012`),
  НЕ развёрнутый блок. Тело/история/доказательства грузятся по ссылке, когда реально нужны.
- Провенанс усиливает это: горячий `feature_list.json` — тонкая проекция (голова), холодный
  append-only лог — источник истины истории (см. `v8-provenance-and-context.md`). Токен-стоимость
  провенанса фиксирована и не растёт с числом правок.

## Enforcement (тест 3 атрибутов)

| Инвариант | Где | Механизм | При обходе |
|---|---|---|---|
| Тело завершённой (архивной) фичи не в горячем | `templates/git-pre-commit.sh` блок 7 | git pre-commit: id архивной фичи (стаб `evidence_hash` / `feature_list.archive.json`) в заголовочной секции CLAUDE.md/SESSION.md с телом ≥3 строк | **warn** (не block — не рвём поток; спека L4-F1) |

**Почему warn, а не block:** разгрузка горячего контекста — гигиена, не корректность. Жёсткий блок
на каждый коммит рвал бы поток из-за эвристики парсинга markdown. Сигнал точечный (только
**архивные** id — фича, уже вынесенная в архив; passing в текущей сессии легитимна до ротации).

## Как разгружать

- **`/checkpoint`** (L4-F2) на естественной границе (конец фичи/волны): ротация завершённых в
  архив через `scripts/archive-features.sh`, перезапись Current State, sync статусов. Управляемо.
- Вручную: `bash scripts/archive-features.sh` — вынести done/superseded/rejected в архив по ссылке.
