# Сначала проверь сам, потом спрашивай

> Источник: реальный случай в CRM-проекте. Основатель-непрограммист: «Ты должен за меня проверить, сам проверь». Урок: пользователь не оператор терминала — агент проверяет инфраструктуру сам.

Пользователь — не техник. Он не выполняет проверки `which docker`, `pgrep -f X`, `lsof -i :PORT`. Это работа агента.

## Запрещённые фразы (regex для самопроверки)

- "убедись что"
- "проверь что"
- "перед запуском проверь"
- "у тебя есть/установлен"
- "запущен ли"
- "авторизован ли"
- "сначала включи"

Эти фразы — **сигнал «сам не проверил»**. Сработали → переписать без них: **сначала bash проверка, потом доклад/действие**.

## Таблица замены

| Хотел спросить | Команда вместо этого |
|---|---|
| «Docker запущен?» | `pgrep -f Docker && docker info` |
| «Установлен Node?» | `which node && node --version` |
| «Установлен Python?» | `which python3 && python3 --version` |
| «Порт 3000 свободен?» | `lsof -i :3000` |
| «PostgreSQL работает?» | `pg_isready` или `lsof -i :5432` |
| «Supabase локально?» | `lsof -i :54321` (default Supabase Studio) |
| «Авторизован в GitHub?` | `gh auth status` ИЛИ `curl -H "Authorization: token $GITHUB_TOKEN" https://api.github.com/user` |
| «Есть ключ X?» | `grep -q '^export X=' ~/.env.shared` |
| «GUI приложение Y установлено?» | `ls /Applications/Y.app` |
| «Git ahead/behind?» | `git status -sb` |
| «Свободное место на диске?» | `df -h .` |
| «Последний коммит?» | `git log -1 --format='%h %ai %s'` |

## Когда МОЖНО спросить пользователя (5 кейсов)

1. **Бизнес-аккаунт** (например доступ к Meta Business, Vercel) — только у пользователя.
2. **Физический ресурс** (домен, реквизиты).
3. **Subjective judgment** про сам продукт.
4. **High blast radius** действия (удалить прод, force-push в main).
5. **Бизнес-выбор** (ICP, pricing, видимая функция).

Во всех остальных случаях — **сначала bash, потом действие**.

## Сигналы что я снова сваливаюсь в чушь-вопросы

- В моём ответе появляется «убедись что...»
- Я говорю «перед запуском проверь...»
- Я предполагаю что у пользователя что-то есть/нет без программной проверки
- Я останавливаюсь и жду от пользователя ответа на технический вопрос, который мог взять сам

При появлении любого сигнала — **стоп**, сам выполнить bash проверку, и продолжить.

## Enforcement

В этом релизе — **дисциплина агента** + правило здесь.

Будущий механизм (если/когда H2 переработан): `agents/quality-gate-validator.md` regex по запрещённым фразам → REWRITE.
