# Stuck Protocol — v5.1

Главное правило: **«3 попытки — STOP. Не 4-я того же.»**

Полное описание команды в `skills/stuck/SKILL.md`. Этот файл — методологический контекст.

---

## Триггеры (auto + manual)

### Auto (из tools-allowlist.yaml)

1. **45 мин без commit/test-pass** → auto `/stuck`
2. **3 неуспешных /verify** на одну фичу
3. **2 одинаковых ошибки подряд** (recurrence в error-journal)
4. **stage-verifier FAIL 3 раза подряд**

### Manual

- Пользователь сказал «в тупике», «крутимся», «не получается»
- Pattern «давай ещё ретрай» больше 2 раз — это сигнал stuck

---

## Шаги протокола

1. **STOP** — не делать 4-ю попытку
2. **Stuck statement** в `docs/stuck-statements/` (5 Why + Ишикава + контекст + что пробовали)
3. **LLM-кворум** 3-4 моделей параллельно (с budget cap $5):
   - Claude через подписку Claude Code (subagent, $0)
   - Gemini CLI (`gemini -p "$(cat statement.md)"`)
   - Codex CLI
   - ChatGPT API (опционально)
4. **Synthesizer** агрегирует ответы → `docs/research/llm-quorum-synthesis-<ts>.md`:
   - Триангулированные гипотезы (несколько LLM предложили одно и то же — высокий приоритет)
   - Уникальные интересные идеи
   - Отвергнутые (уже пробовали)
   - 3 подхода для пользователя
5. **Эскалация пользователю** в формате business-impact (НЕ technical A/B):
   - A/B/C подхода с временем/риском/качеством одной фразой
   - Рекомендация системы + причина
6. **Новый подход** — обнулить counter, не продолжать старый

---

## Лимиты

- **3 stuck-протокола на фичу** — если третий, `/escalate-to-human` (остановка плагина, серьёзный разговор)
- **Stuck-statement НИКОГДА не удаляется** — архив в docs/stuck-statements/ (история решений)
- **Кворум минимум 2 LLM** — одна модель = одна точка зрения

---

## Privacy в кворуме

Перед отправкой stuck-statement во внешние LLM:
- Вырезать `<private>` теги
- Замаскировать ключи, эндпоинты, ИНН клиентов
- Не отправлять файлы с секретами

---

## Что НЕ stuck (прямая эскалация без протокола)

- Отсутствие данных (нет ключа, нет доступа) — ASK напрямую
- Инфра-блокер (нет интернета) — ASK
- Бизнес-вопрос (что важнее A или B) — ASK (с Quality Gate format)

Stuck-протокол — **только алгоритмический/архитектурный** тупик.

---

## Связь с подсистемой Learning

Каждый stuck-event:
- → запись в `error-journal.md` как класс ошибки
- → после решения — урок промоутится в `feedback_*.md`
- → если корневая причина повторилась — система обновляет `~/CLAUDE.md` (с confirm пользователя)

---

## Отличия от других эскалаций

| Ситуация | Что делаем |
|---|---|
| 1 неуспешная попытка | Корректировка, ещё раз |
| 2 неуспешных | Значимая корректировка |
| 3 неуспешных | **Stuck-протокол** |
| Нет данных | ASK напрямую |
| Инфра блокер | ASK напрямую |
| Бизнес-выбор | ASK через Quality Gate |
