# headroom — осознанно НЕ в ядро (v8 L4-F6, эксперимент)

> Решение владельца: «осторожно, не гнаться за экономией, не терять данные». Берём только
> неразрушающий датчик — и только под флагом в одноразовой ветке, НЕ в основной код плагина.

## Почему headroom НЕ в ядро

headroom (внешний прокси, ужимающий контекст на лету) оптимизирует **другую ось**, чем v8:

- **Оптимизирует цену за вызов**, а не durable state. Наша цель (Линия 4) — малый горячий
  контекст + состояние в файлах под управлением; это уже снимает бо́льшую часть экономии **без
  риска**. Файл-first подход и `/checkpoint` (L4-F2) решают задачу надёжнее.
- **Риск потери данных** через TTL-кэш ровно там, где владелец просил НЕ терять: авто-эвикция
  или обрезка контекста может выкинуть ход, который ещё нужен.
- **Внешний proxy в headless-автозапуске** — лишняя движущаяся часть в цепочке, где нет человека,
  чтобы заметить сбой (эстафета сессий, cron).

Поэтому в enforcement-счёт L4-F6 **НЕ входит** (честность деклараций). Это не механизм — это
осознанный отказ + контракт на случай будущего эксперимента.

## Контракт эксперимента (если кто-то захочет замерить)

Промоушн headroom из «нет» в «может быть» разрешён ТОЛЬКО при соблюдении ВСЕХ условий:

1. **Только читающая часть.** Датчик measures usage → warn → триггерит `/checkpoint`. НИКАКОЙ
   авто-эвикции / авто-обрезки / авто-сжатия контекста. Любая разрушающая операция — **reject**.
2. **Под флагом в одноразовой ветке / тест-проекте**, не в основном коде плагина.
3. **Доказательство перед промоушном:** после N сессий замера — показать, что **ни один ход
   контекста не был удалён автоматически** (лог всех решений датчика + сверка с транскриптом).
4. **Sunset-контракт:** ветка эксперимента с рождения несёт запись «зачем / при каком условии
   умирает / как демонтируется». Не приживётся — удаляется, не обрастает кодом (временные леса без
   плана демонтажа становятся несущей конструкцией).

Пока эти условия не выполнены — headroom не существует в продукте. Экономию контекста несут
файл-first состояние (провенанс + архив по ссылке) и управляемый `/checkpoint`.
